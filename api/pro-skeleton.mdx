# ProSkeleton API

ProSkeleton provides loading skeleton screens for Pro components, matching the layout of ProTable, ProList, and ProDescriptions.

## ProSkeletonProps

<ParamField path="type" type="'list' | 'descriptions' | 'result'" default="list">
  Type of skeleton to display.
  
  - `list` - Skeleton for ProTable/ProList
  - `descriptions` - Skeleton for ProDescriptions
  - `result` - Skeleton for Result page
</ParamField>

<ParamField path="active" type="boolean" default="true">
  Whether to show animation.
</ParamField>

## List Skeleton Props

When `type="list"`:

<ParamField path="pageHeader" type="boolean | PageHeaderSkeletonProps" default="true">
  Whether to show page header skeleton.
  
  Can be an object to customize:
  ```typescript
  {
    active?: boolean;
  }
  ```
</ParamField>

<ParamField path="toolbar" type="boolean | ToolbarSkeletonProps" default="true">
  Whether to show toolbar skeleton.
  
  Can be an object to customize:
  ```typescript
  {
    active?: boolean;
    actions?: number; // Number of action buttons
  }
  ```
</ParamField>

<ParamField path="list" type="boolean | ListSkeletonProps" default="true">
  Whether to show list skeleton.
  
  Can be an object to customize:
  ```typescript
  {
    active?: boolean;
    size?: 'default' | 'large' | 'small';
    rows?: number; // Number of list items
  }
  ```
</ParamField>

<ParamField path="statistic" type="boolean | number" default="false">
  Whether to show statistic cards.
  
  - `boolean` - Show/hide statistics
  - `number` - Number of statistic cards to show
</ParamField>

## Descriptions Skeleton Props

When `type="descriptions"`:

<ParamField path="pageHeader" type="boolean" default="false">
  Whether to show page header skeleton.
</ParamField>

<ParamField path="list" type="boolean | number" default="true">
  Whether to show descriptions skeleton.
  
  - `boolean` - Show/hide descriptions
  - `number` - Number of description items to show
</ParamField>

<ParamField path="columns" type="number" default="3">
  Number of columns for descriptions.
</ParamField>

## Usage Examples

<Tabs>
  <Tab title="List Skeleton">
    ```tsx
    import { ProSkeleton } from '@ant-design/pro-components';

    // Full page skeleton with header, toolbar, and table
    <ProSkeleton type="list" />

    // Customized list skeleton
    <ProSkeleton
      type="list"
      pageHeader={{
        active: true,
      }}
      toolbar={{
        active: true,
        actions: 3,
      }}
      list={{
        active: true,
        rows: 10,
        size: 'large',
      }}
      statistic={4}
    />

    // Table only (no header or toolbar)
    <ProSkeleton
      type="list"
      pageHeader={false}
      toolbar={false}
    />
    ```
  </Tab>
  <Tab title="Descriptions Skeleton">
    ```tsx
    import { ProSkeleton } from '@ant-design/pro-components';

    // Basic descriptions skeleton
    <ProSkeleton type="descriptions" />

    // With page header and custom columns
    <ProSkeleton
      type="descriptions"
      pageHeader
      list={12}
      columns={2}
    />
    ```
  </Tab>
  <Tab title="Result Skeleton">
    ```tsx
    import { ProSkeleton } from '@ant-design/pro-components';

    // Result page skeleton
    <ProSkeleton type="result" active />
    ```
  </Tab>
  <Tab title="With Loading State">
    ```tsx
    import { ProTable, ProSkeleton } from '@ant-design/pro-components';
    import { useState, useEffect } from 'react';

    export default () => {
      const [loading, setLoading] = useState(true);
      const [data, setData] = useState([]);

      useEffect(() => {
        fetchData().then(result => {
          setData(result);
          setLoading(false);
        });
      }, []);

      if (loading) {
        return <ProSkeleton type="list" />;
      }

      return (
        <ProTable
          dataSource={data}
          columns={columns}
        />
      );
    };
    ```
  </Tab>
  <Tab title="Individual Components">
    ```tsx
    import {
      PageHeaderSkeleton,

      DescriptionsSkeleton,

    // Build custom skeleton layout
    ```

  </Tab>

## Best Practices

1. **Match component type**: Use `type="list"` for ProTable/ProList and `type="descriptions"` for ProDescriptions to match the actual component layout.

2. **Show during data loading**: Display skeleton while fetching data from API to improve perceived performance.

3. **Customize for your layout**: Adjust `rows`, `columns`, and other props to match your actual component configuration.

4. **Keep animations**: The default `active` animation provides better visual feedback.

5. **Hide unnecessary parts**: Set `pageHeader={false}` or `toolbar={false}` if your component doesn't have those sections.
